Section 1:        Overview of Assessment

The assignment is worth 40% of the overall mark for the challenges, which make up 75% of the modules overall mark.

The assignment is aimed at introducing embedded development with the BBC micro:bit.

The assignment is described in more detail in section 2.

This is a individual assignment.

Working on this assignment will help you to understand the Internet of Things, embedded systems, and programing with C++. If you have questions about this assignment, please post them to the discussion board hosted on Blackboard.

Challenge 2 must be submitted via, with the completion of a challenge 2 submission form, found on Blackboard, by 2pm on the 1st March, 2018.

Section 2:        Task Specification

Implement a simple communication protocol that transmits data between 2 BBC Micro:bit devices, using GPIOs.

Your solution should be implemented using C++.

Your solution must be committed in UWE's Gitlab. It will naturally be time stamped and you must be careful to not make commits after the submission deadline.

The basic protocol, defined below, is Morse code, however, implementing these bare requirements will enable you to reach a maximum of 70%. To get beyond the 70% your protocol should be more, for example you might choose to implement the transmission of characters messages that is encrypted with a simply cypher.

Your solution's source code should follow a coding convention, it should be well commented, and include a README.md on how to build it and what and how to use your solution.

This challenge can be done individually or in groups of a MAXIMUM size of 2, i.e. pairs. Submission should be individually and the README.md must document who contributed to the submission.

Morse Code (https://en.wikipedia.org/wiki/Morse_code)

Morse code is an approach to transmitting text information as a series of on-off-tones across a telegraph wire. It is an example of an encoding and a protocol, all be it a very simple one. The encoding is shown on the left.

For this challenge you are expected to utilize two Micro:bits to transmit Morse code. A single button, on each Micro:bit, should be used to input data to be transmitted, a single wire is used to transmit messages between the Micro:bits, and the LED matrix displays the sent and received messages. The sender will display either a dash or dot, as the message is entered, while the receiver should display the decoded text. A demonstration of this configuration is given below.

An important aspect to consider is how to synchronize the two Micro:bits. The simplest approach to this is to time messages, e.g. a dot might be recorded if the GPIO line is held high for 250ms, while a dash would be recorded if the line is high for more than 250ms, but less that 500ms. Anything else is considered noise, i.e. not a message.

Section 3:        Deliverables

  • The challenge solution, e.g. C++ implementation, should be committed in its own directory in UWE's Gitlab, including a README.md documenting what the program does, how to build it, and any additional documents).
  • The source code must consist of more than a single source file, including at least one header file, and two .cpp files.
  • Your submission should include, in the README.md, a description of your protocol, including a state diagram of its working. If implement an alterative protocol that uses encryption, then your submission should include details of this and how it works.
  • Submit a signed PDF submission form to Blackboard, which should include a link to your Git project for this module.
